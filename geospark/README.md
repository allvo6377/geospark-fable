# GeoSpark Onestop Solution — Website

Marketing site + content management system for GeoSpark Onestop Solution (Nairobi, Kenya).
Built with **Next.js 16** (App Router, server-rendered & statically cached — excellent SEO and speed) and **Payload CMS 3** embedded in the same app, backed by SQLite.

## Quick start

```bash
npm install
npm run seed     # first time only — fills the CMS with all content + creates the admin user
npm run dev      # http://localhost:3000  (site)  ·  http://localhost:3000/admin  (CMS)
```

First admin login: `admin@geospark.co.ke` / `GeoSpark-Admin-2026!` — **change this password immediately** (Admin → Users).

## Editing the website (no code needed)

Everything visible on the site is editable at **`/admin`**:

| What you want to change | Where in the admin panel |
| --- | --- |
| Logo, phone, email, WhatsApp, nav links, footer, the dark CTA banner | **Globals → Site Settings** |
| Any text on Home / Services / About / Projects / Contact / Blog pages | **Globals → Page: …** (one entry per page, grouped in tabs) |
| The six service cards | **Collections → Services** |
| Team members & their photos | **Collections → Team Members** |
| Client quotes | **Collections → Testimonials** |
| Portfolio projects (cards + map pins) | **Collections → Projects** (lat/lng controls the map marker) |
| Blog articles | **Collections → Posts** (save as *draft* first, then publish; tick **featured** for the big top slot) |
| Blog categories | **Collections → Categories** |
| Images | **Collections → Media** (upload once, reuse anywhere) |
| Contact-form messages from visitors | **Collections → Submissions** (read-only inbox) |

Changes appear on the live site within **2 minutes** (pages are cached for speed and refresh automatically).

## Stack & why

- **Next.js 16 App Router** — server-rendered HTML with static caching: fast first paint, full SEO (per-page metadata, sitemap at `/sitemap.xml`, robots at `/robots.txt`, JSON-LD structured data).
- **Payload CMS 3** — lives inside the same Next.js app at `/admin`. No third-party CMS subscription, content is queried directly from the database at render time (no HTTP hop).
- **SQLite** — zero-config, file-based (`geospark.db`). Can be swapped for Postgres by changing one adapter in `src/payload.config.ts` when scale demands it.
- **sharp** — automatic image resizing (thumbnail / card / hero renditions) for every upload.
- **Leaflet + OpenStreetMap** — the project and contact maps, no API key required.
- **next/font** — Space Grotesk, Instrument Sans and IBM Plex Mono self-hosted (no render-blocking Google Fonts request).

All animations from the design handoff are preserved: scroll reveals, magnetic buttons, cursor-following ambient light, page-transition veil, animated counters, technology marquee, pulsing map markers, skeleton “FETCHING TILE” staggers, FAQ accordion, dark/light theme with saved preference, and `prefers-reduced-motion` support throughout.

## Security notes

- Admin panel is authenticated (Payload auth, bcrypt-hashed passwords, CSRF-protected).
- Public API access is read-only for site content; contact submissions can only be **created** through the server action (with honeypot spam protection) and only **read** by logged-in users.
- `PAYLOAD_SECRET` lives in `.env` (never commit it — `.gitignore` already excludes it).
- Uploads restricted to images.

## Deploying

The app is a standard Node/Next.js server:

```bash
npm run build
npm start        # serves on PORT (default 3000)
```

Host on any provider with a **persistent disk** (Railway, Render, Fly.io, a VPS with PM2/systemd, etc.) so `geospark.db` and the `media/` folder survive restarts. Set these environment variables in production:

```
PAYLOAD_SECRET=<long random string>
DATABASE_URL=file:./geospark.db
NEXT_PUBLIC_SERVER_URL=https://your-domain.co.ke
```

> Vercel note: Vercel’s filesystem is ephemeral. To deploy there, switch the DB adapter to `@payloadcms/db-postgres` (e.g. Neon) and media storage to `@payloadcms/storage-vercel-blob` — both are drop-in changes in `src/payload.config.ts`.

## Project layout

```
src/
  payload.config.ts        # CMS schema entry point
  collections/             # Posts, Services, Projects, Team, Media, …
  globals/                 # Site Settings + one editable global per page
  app/(frontend)/          # the public website (pages + globals.css)
  app/(payload)/           # the /admin panel (generated by Payload)
  components/              # Nav, Footer, CTA banner, maps, blog, animation fx
  seed/                    # `npm run seed` — initial content
```
